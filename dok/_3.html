Архітектура шаблону. Файли та папки

Розархівувавши завантажений шаблон, в корені ви виявите такі файли та папки:

👉 Файл package.json – файл, що містить команди запуску збирачів, інформацію про встановлені плагіни та їх версії (можна редагувати за потреби). А також іншу інформацію про автора та версію шаблону.

👉 Файл gulpfile.js – головний файл збирача GULP. Містить підключення різних завдань та сценарії їх виконання (можна редагувати за потреби)

👉 Файл snnipets.txt – файл зі сніппетами, які використовуються при роботі з шаблоном. Як їх додати до свого редактора

👉 Файл README.txt – коротка інформація щодо роботи з шаблоном

👉 Файл cover.webp – обкладинка ЧФ

👉 Папка config – містить папки та файли для налаштування роботи інструментів, а також файли з окремими завданнями.

👉 Папка src – містить вихідні файли та папки проекту.

Вміст папки config:

👉 Файл gulp-settings.js – містить налаштування шляхів до файлів та папок для роботи GULP, а також налаштування FTP з’єднання для вивантаження результату на сервер

👉 Файл gulp-plugins.js – допоміжний файл, містить підключення та експорт списку загальних плагінів для завдань GULP

👉 Файл webpack.dev.js – містить конфігурацію роботи WEBPACK у режимі розробника

👉 Файл webpack.prod.js – містить конфігурацію роботи WEBPACK у режимі продакшн

👉 Папка gulp-tasks – містить файли окремих завдань GULP

Вміст папки src:

👉 Файл favicon.ico – іконка сайту

👉 Файл index.html -індексна сторінка-зміст, вона не оновлюється автоматично під час редагування (за бажанням видалити)

👉 Файл home.html – головна сторінка сайту (за бажанням перейменувати в index.html)

👉 Папка files (порожня) – всі файли цієї папки будуть скопійовані в папку з результатом (dist/files)

👉 Папка fonts (порожня) – використовується для підключення локальних та іконкових шрифтів

👉 Папка svgicons (порожня) – використовується для створення SVG спрайту

👉 Папка html – містить .HTM файли, які підключаються до HTML файлів сторінок (наприклад, home.html). Тут можна створювати свої файли, що підключаються

👉 Папка scss – містить файли та папки стилів сайту

👉 Папка js – містить файли та папки скриптів сайту

👉 Папка img – тут зберігаються зображення проекту. Спочатку тільки кавер-фото шаблону

Вміст папки src/html:

Файли цієї папки призначені для підключення до HTML-файлів сторінок.

👉 Файл _header.htm – підготовка до створення шапки сайту. Містить тег header та обмежуючий контейнер. Спочатку підключено до home.html

👉 Файл _footer.htm – підготовка до створення підвалу сайту. Містить тег footer та обмежуючий контейнер. Спочатку підключено до home.html

👉 Файл _head.htm – містить тег head всередині якого вказані метатеги, підключена іконка сайту, файл стилів, а також тег title зізмінною-заголовком. Спочатку підключено до home.html

👉 Файл _js.htm – призначений для підключення загальних JS-файлів. Містить підключення основного файлу js/app.min.js із результату. Спочатку підключено до home.html

👉 Файл _popup.htm – підготовка до створення попапов. Містить шаблонний закоментований HTML-код. Спочатку підключений до home.html.

👉 Файл _pagging.htm – підготовка до створення блоку посторінкової навігації. Містить шаблонний HTML-код.

👉 Файл _preloader.htm – містить HTML CSS та JS функціонал

Вміст папки src/scss:

👉 Файл style.scss – основний файл стилів. Містить різні налаштування та підключення інших файлів.

👉 Файл home.scss – файл стилів на головній сторінці. Спочатку порожній і підключений до style.scss

👉 Файл header.scss – файл стиль шапки сайту. Містить закоментований код кнопки меню-бургера. Спочатку підключено до style.scss

👉 Файл footer.scss – файл стилів підвалу сайту. Спочатку порожній і підключений до style.scss

👉 Файл common.scss – файл стилів для загальних блоків конкретного проекту, що перевикористовуються. Спочатку порожній і підключений до style.scss

👉 Файл base.scss – файл базового стилю шаблону ЧФ. Містить корисні SCSS-шаблони та закоментовані підключення файлів із базовими стилями різних модулів.

👉 Папка libs – містить файли повних (заводських) стилів різних модулів та плагінів

👉 Папка fonts – містить файл(и) стилів, пов’язаних зі шрифтами. Містить файл icons.scss для підключення іконочного шрифту. Також,, у процесі підключення локальних шрифтів, тут з’являється файл fonts.scss

👉 Папка – містить допоміжні файли стилів ЧФ, також файли з базовими стилями різних моделей. Для зручності файли модулів однієї групи (наприклад елементи форм) зібрані в окремі підпапки. Тут же знаходиться файл із обнуляючими стилями, null.scss

Вміст папки src/js:

👉 Файл app.js – основний файл скриптів. Служить для підключення необхідного у проекті функціоналу та інших налаштувань.

👉 Папка libs – містить файли готових доповнень, як правило, не передбачені для редагування.

👉 Папка files – містить файли різних модулів та іншого функціоналу.

Вміст папки src/js/files:

👉 Файл script.js – призначений для написання свого коду для проекту. Спочатку імпортовано до app.js

👉 Файл functions.js – містить дрібні модулі та різний допоміжний функціонал. Спочатку імпортований в app.js, але модулі закоментовані

👉 Файл gallery.js – містить підключення плагіна та стилів галереї. Спочатку імпортований в app.js, закоментований. При необхідності, в нього можна вносити зміни

👉 Файл sliders.js – містить підключення плагіна та стилів слайдера, функціонал для автоматичного додавання класів, код для створення конкретних слайдерів. Спочатку імпортований в app.js, закоментований.

👉 Файл tippy.js – містить підключення плагіна та стилів плагіна підказок. Спочатку імпортований в app.js, закоментований.

👉 Файл map.js (в работі) – містить шаблонний код для реалізації функціоналу карток.

👉 Файл modules.js – допоміжний файл для роботи ЧФ.

👉 Папка forms – містить файли модулів для роботи з формами.

👉 Папка scroll – містить модулі для роботи з прокручуванням сайту.
